## 特征交叉神经网络的训练过程

特征交叉神经网络（如 DeepFM, NFM, DCN, PNN 等）的训练过程本质上是一个**端到端（End-to-End）的监督学习过程**。

尽管模型内部结构复杂（包含 FM、线性部分、DNN 等），但在训练层面，它们都遵循标准的深度学习训练范式：**前向传播计算 Loss $\to$ 反向传播计算梯度 $\to$ 优化器更新参数**。

以下是训练流程的详细解析，特别关注**联合训练（Joint Training）\**和\**Embedding 更新**这两个核心点。

------

### 1. 训练目标与损失函数 (Loss Function)

绝大多数特征交叉网络用于推荐系统中的 **CTR 预估（点击率预测）**。

- **输入**：用户的特征（ID 类、数值类）。
- **标签 ($y$)**：用户是否点击（0 或 1）。
- **输出 ($\hat{y}$)**：预测的点击概率（0 到 1 之间的浮点数，经 Sigmoid 输出）。

因此，最常用的损失函数是 **LogLoss (对数损失)**，也称为**二元交叉熵损失 (Binary Cross Entropy)**：
$$
Loss = - \frac{1}{N} \sum_{i=1}^{N} \left[ y_i \log(\hat{y}_i) + (1 - y_i) \log(1 - \hat{y}_i) \right] + \lambda \| \Theta \|^2
$$

- 第一部分衡量预测概率与真实点击行为的差距。
- 第二部分 $\lambda \| \Theta \|^2$ 是 **L2 正则化项**，用于防止过拟合（特别是防止 Embedding 参数过拟合）。

------

### 2. 核心机制：联合训练 (Joint Training)

这是 DeepFM 等模型训练的关键。模型中包含两部分：

1. **浅层/交叉组件**（如 FM, Linear）：负责记忆和低阶交叉。
2. **深层组件**（如 DNN）：负责泛化和高阶交叉。

**“联合训练”意味着这两部分同时进行训练，并且它们共享底层的 Embedding 矩阵。**

#### 梯度的反向传播 (Backpropagation)

当计算出 Loss 后，梯度会从输出层向后传递。对于 **Shared Embedding Matrix**（最底层的参数矩阵 $\mathbf{V}$），它会**同时收到两股梯度**：
$$
\frac{\partial Loss}{\partial \mathbf{V}} = \underbrace{\frac{\partial Loss}{\partial \text{Output}} \cdot \frac{\partial \text{Output}}{\partial \text{FM}} \cdot \frac{\partial \text{FM}}{\partial \mathbf{V}}}_{\text{来自 FM 侧的梯度}} + \underbrace{\frac{\partial Loss}{\partial \text{Output}} \cdot \frac{\partial \text{Output}}{\partial \text{DNN}} \cdot \frac{\partial \text{DNN}}{\partial \mathbf{V}}}_{\text{来自 DNN 侧的梯度}}
$$

- **直观理解**：Embedding 向量不仅要调整自己以适应 FM 的二阶内积规则（让共现的特征向量更相似或内积更大），还要调整自己以适应 DNN 的高阶非线性变换。
- **结果**：这种双重监督使得学习到的 Embedding 表示更加精准和鲁棒。

------

### 3. 优化算法 (Optimizer)

在推荐场景下，特征非常稀疏（Sparse），这意味着对于一个 batch 的数据，只有极少数的 Embedding 向量会被激活和更新。

因此，**SGD（随机梯度下降）通常不是最佳选择**。最常用的优化器是 **Adam** 或 **Adagrad**。

- **自适应学习率**：Adam 可以为每个参数（即每个特征的 Embedding 向量中的每个元素）独立调整学习率。
- **处理稀疏性**：对于频次低的特征（更新机会少），Adam 会给较大的学习率让它快速收敛；对于频次高的特征（更新机会多），给较小的学习率。

------

### 4. 完整的训练步骤跟踪

假设我们要训练一个 DeepFM 模型，Batch Size = 256。

#### Step 1: Batch Data 采样

从训练集中取出一个 Batch（256 条样本）。每条样本包含用户的特征索引（如 `[User_ID=5, Item_ID=10, ...]`) 和标签（`Click=1`）。

#### Step 2: 前向传播 (Forward Pass)

1. **查表**：根据特征索引，从 Embedding 矩阵中取出对应的向量。
2. **分流**：
   - **路 A (FM)**：计算一阶和二阶内积。
   - **路 B (DNN)**：将向量展平，通过 ReLU 全连接层。
3. **融合**：$Logits = \text{Linear} + \text{FM} + \text{DNN}$。
4. **预测**：$\hat{y} = \text{Sigmoid}(Logits)$。

#### Step 3: 计算损失 (Compute Loss)

将预测值 $\hat{y}$ 与真实标签 $y$ 代入 LogLoss 公式计算误差。

#### Step 4: 反向传播 (Backward Pass)

1. 计算 Loss 相对于权重的梯度。
2. **关键点**：DNN 的权重、FM 的权重、以及**底层的 Embedding 矩阵**都会计算出梯度。

#### Step 5: 参数更新 (Update)

优化器（Adam）根据计算出的梯度，减去（学习率 $\times$ 梯度），更新模型参数。

*注意：对于 Embedding 矩阵，只有当前 Batch 中出现的那些特征对应的向量会被更新，没出现的特征保持不变。*

------

### 5. 防止过拟合的特殊手段

由于推荐数据极其稀疏且维度极高（特征可能上亿），特征交叉网络非常容易过拟合（记住了训练集的数据，但无法泛化）。训练时通常必须加入：

1. **L2 正则化**：对 Embedding 层的权重施加惩罚，防止向量数值过大。
2. **Dropout**：在 DNN 层之间随机“丢弃”部分神经元，强迫网络学习鲁棒的特征，而不是依赖某几个特定的神经元。
3. **Early Stopping**：监控验证集（Validation Set）的 Loss，一旦验证集 Loss 不再下降（开始上升），就停止训练。

### 总结

特征交叉网络的训练是一个**端到端**的过程，其核心在于**通过 LogLoss 引导，利用 Adam 优化器，让底层的共享 Embedding 层在 FM（浅层交互）和 DNN（深层交互）的双重梯度指导下，学习到最优的特征表示。**

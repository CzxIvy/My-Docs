尽管这些模型（FM, DeepFM, NFM, PNN 等）内部计算“特征交叉”的方式各不相同（内积、外积、元素积、MLP 等），但在**最顶层的输入**和**最底层的输出**上，它们是非常统一的。

它们都属于**点击率预测 (CTR Prediction)** 或**推荐排序**模型。

------

### 1. 最开始的输入 (Input)

模型的输入通常是**稀疏的特征数据**，主要分为两类：

#### A. 类别特征 (Categorical Features)

这是推荐系统中最主要的数据。

- **示例**：用户 ID (`User_1001`)、电影 ID (`Movie_Avatars`)、城市 (`Beijing`)。
- **输入形式**：
  - **概念上**：高维稀疏的 **One-Hot 向量**。例如，如果总共有 100 万个用户，`User_1001` 就是一个长度为 100 万的向量，只有一位是 1，其余是 0。
  - **工程实现上**：通常输入的是特征的 **ID 索引 (Index)**。例如 `[User_ID=45, City_ID=8, Gender_ID=1]`。

#### B. 数值特征 (Numerical Features)

- **示例**：年龄 (`25`)、价格 (`99.9`)、历史点击率 (`0.5`)。
- **输入形式**：直接是实数值（Scalar）。
- *注意：在深度模型中，数值特征有时也会被离散化（分桶）变成类别特征，或者直接作为 Dense 特征输入。*

------

### 2. 核心处理流程 (The Black Box)

虽然输入输出一样，但中间发生了什么？

1. **Embedding 层 (所有模型共有)**：
   - 输入 $\to$ 查表 (Lookup) $\to$ **稠密隐向量 (Dense Embeddings)**。
   - 这是特征交叉的基础。
2. **特征交叉层 (各模型不同)**：
   - **FM**：做内积 $\langle \mathbf{v}_i, \mathbf{v}_j \rangle$。
   - **NFM**：做元素积 $\mathbf{v}_i \odot \mathbf{v}_j$ 然后进 MLP。
   - **IPNN**：做内积求和再 L2 范数，进 MLP。
   - **DeepFM**：FM 部分做内积 + Deep 部分直接拼接进 MLP。

------

### 3. 最终的输出 (Output)

无论中间怎么折腾，最终输出通常只有一个数值。

#### A. 形式

- **标量 (Scalar)**：一个实数。
- **范围**：通常在 **0 到 1 之间**。

#### B. 物理含义

- **点击率概率 (Predicted Probability)**：表示用户点击该物品的可能性。
  - `0.8` 代表有 80% 的概率会点击。
  - `0.1` 代表只有 10% 的概率会点击。

#### C. 激活函数

为了将模型计算出的任意实数（Logits，范围 $(-\infty, +\infty)$）压缩到 $[0, 1]$ 区间，最后一层通常会经过 **Sigmoid 函数**：
$$
\hat{y} = \text{Sigmoid}(logits) = \frac{1}{1 + e^{-logits}}
$$

------

### 总结示例

假设我们要预测**用户 A (User_A)** 是否会点击 **运动鞋 B (Item_B)**。

| **阶段**         | **数据形式**   | **内容示例**                                                 |
| ---------------- | -------------- | ------------------------------------------------------------ |
| **1. 原始输入**  | 稀疏特征 / ID  | `[User_ID: 1024, Item_ID: 55, Category: Shoes]`              |
| **2. Embedding** | 稠密向量       | User: `[0.1, -0.5, ...]` Item: `[0.8, 0.2, ...]`             |
| **3. 模型处理**  | 特征交叉       | **FM**: 计算向量内积 **PNN**: 计算投影后的 L2 范数 **NFM**: 计算向量元素积 |
| **4. Logits**    | 实数 (Score)   | `2.5` (综合了线性项、交叉项、深度项的分数)                   |
| **5. 最终输出**  | **概率 (0~1)** | **`0.924`** (经过 Sigmoid，表示极大概率会点击)               |

一句话总结：

输入是稀疏的 ID，输出是点击的概率。中间的所有复杂网络（FM, PNN, NFM）都是为了更准确地拟合这个概率。
